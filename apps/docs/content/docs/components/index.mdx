Kairo ships 28 components across five categories: forms, navigation, display,
disclosure and overlay. Every interactive component is a thin wrapper over a
Base UI primitive, styled entirely with the vanilla CSS shipped in
`@kairo-ui/theme` — no Tailwind or CSS-in-JS runtime required in your app.
Browse a category below, or jump straight to a component.

## Forms

| Component                                          | Description                                                                                                              |
| --------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| [Button](/docs/components/button)                   | A clickable action element with solid, outline and ghost variants in three sizes.                                          |
| [Input](/docs/components/input)                     | A native text input styled with Kairo's theme, supporting disabled and invalid states.                                     |
| [NumberField](/docs/components/number-field)         | A locale-aware numeric input with increment/decrement stepper buttons, built on Base UI.                                    |
| [Select](/docs/components/select)                   | A dropdown listbox for choosing a value from a predefined list, built on Base UI.                                           |
| [Combobox](/docs/components/combobox)               | A text input paired with a filtered listbox for picking a value from a searchable list, built on Base UI.                  |
| [Checkbox](/docs/components/checkbox)               | A tri-state (checked/unchecked/indeterminate) control, built on Base UI.                                                    |
| [RadioGroup](/docs/components/radio-group)           | A set of mutually exclusive options, exactly one of which can be selected, built on Base UI.                                |
| [Switch](/docs/components/switch)                   | A toggle control for binary on/off state, built on Base UI.                                                                 |
| [ToggleGroup](/docs/components/toggle-group)         | A two-state button that stays pressed, standalone or grouped into a single- or multi-select ToggleGroup, built on Base UI. |
| [Slider](/docs/components/slider)                   | A draggable control for selecting a numeric value, or a range of two, along a track, built on Base UI.                     |

## Display

| Component                                | Description                                                                                                                                                       |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Badge](/docs/components/badge)             | A small label for statuses, counts and metadata, with solid, soft and outline variants across five intents.                                                        |
| [Avatar](/docs/components/avatar)           | Displays a user's profile picture, with an automatic fallback to initials or text, built on Base UI.                                                               |
| [Card](/docs/components/card)               | A container for grouping related content, composed from Header, Title, Description, Content and Footer parts.                                                      |
| [Spinner](/docs/components/spinner)         | A circular loading indicator that adapts to the surrounding text color.                                                                                             |
| [Separator](/docs/components/separator)     | A thin rule that visually or semantically divides content, with horizontal and vertical orientations.                                                              |
| [Progress](/docs/components/progress)       | A static task-completion bar with determinate and indeterminate states, built on the WAI-ARIA progressbar pattern.                                                 |
| [Meter](/docs/components/meter)             | A static gauge for a known, bounded quantity — disk usage, password strength, storage quota — with low/high/optimum thresholds that tint the fill automatically. |
