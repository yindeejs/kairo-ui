# Security Policy

## Supported Versions

Kairo is currently pre-1.0 (`0.x`). Until the `v1.0.0` release, only the
**latest published `0.x` version** of `@kairo-ui/react` and `@kairo-ui/theme`
receives security fixes — there is no long-term support branch yet.

| Version      | Supported          |
| ------------ | ------------------ |
| 0.x (latest) | :white_check_mark: |
| < latest 0.x | :x:                |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report it privately using GitHub's security advisory feature:

**https://github.com/yindeejs/kairo-ui/security/advisories/new**

This opens a private conversation with the maintainers so the issue isn't
disclosed publicly before a fix is available.

Please include as much detail as you reasonably can:

- The affected package(s) and version(s) (`@kairo-ui/react`, `@kairo-ui/theme`)
- Steps to reproduce, or a minimal repro
- The potential impact as you understand it

## What to Expect

- We'll acknowledge new reports as soon as we can.
- We'll investigate and let you know whether it's confirmed as a
  vulnerability, and roughly how we plan to address it.
- Once a fix is released, we'll credit the reporter (unless you'd prefer to
  stay anonymous) and publish a GitHub Security Advisory describing the
  issue.

Thanks for helping keep Kairo and its users safe.
